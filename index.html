ليه الـ AI جوا الـ Flow؟

تأسيس الدافع قبل الشرح التقني

في المحاضرة الرابعة بنينا Flow بينقل البيانات من الإيميل للـ Excel. بس الـ Flow كان "بيلم" — بينقل النص كما هو بدون ما يفهمه. النهاردة هنخلي الـ Flow "يفكر" — يستلم الإيميل، يفهمه، يلخصه، ويستخرج البيانات المهمة قبل ما يحفظها.

الأتمتة العادية تنقل البيانات. الأتمتة الذكية تفهم البيانات. الفرق بينهم هو إنك بتوفر مش بس وقت النقل، لكن وقت القراءة والتحليل كمان.

  • مش بيفهم المحتوى
  • بيحفظ كل الكلام (حتى غير المهم)
  • محتاج تدخل بشري للتلخيص

٠٣

HTTP Action و الـ APIs

المفهوم الأساسي — ٥ دقيقة

عشان Power Automate يكلم Claude، محتاجين "مترجم". المترجم ده اسمه HTTP Action. والـ API هو "الباب" اللي بنطرق عليه عشان نكلم الـ AI.

الـ API = الباب. الـ HTTP Action = الجرس. والـ JSON = لغة الطلب.
لو فهمت التشبيه ده، فهمت 90% من دمج الـ AI مع الأتمتة.

مثال ١

تشبيه المطعم

في المطعمفي Power Automateالوظيفة
الزبونPower Automateاللي عنده الطلب (الإيميل)
الجارسونHTTP Actionياخد الطلب ويوديه للمطبخ
المطبخClaude APIبيطبخ الطلب (بيحلل النص)
قائمة الطلبJSON Bodyلغة منظمة يفهمها المطبخ
الطبقJSON Responseالنتيجة المنظمة اللي ترجع
API
Application Programming Interface. واجهة برمجية بتخلي التطبيقات تتكلم مع بعض. Claude API = الباب اللي بنبعتله النص وهو بيرجعلنا الرد المنظم.
HTTP Action
إجراء HTTP. Action في Power Automate بيبعت طلب (Request) لعنوان URL معين، ويستقبل الرد (Response). ده اللي بيخلي الـ Flow يكلم أي API في العالم.
JSON
JavaScript Object Notation. لغة نصية منظمة بتستخدمها الـ APIs. شكلها: {"name": "Ahmed", "age": 30}. سهلة القراءة للإنسان والآلة.
Endpoint
نقطة النهاية (URL). العنوان اللي بنبعتهله الـ Request. مثال لـ Claude: https://api.anthropic.com/v1/messages
نصيحة عملية

متخافش من كلمة API. هي مجرد "عنوان ويب" + "مفتاح دخول" + "رسالة منظمة". لو تعرف تبعت إيميل، تقدر تبعت API Request.

٠٤

الإعدادات: API Key + Cost Warning

تجهيز البيئة قبل البناء — ١ دقيقة

قبل ما نبني الـ Flow، لازم نجهز "مفتاح الدخول" (API Key) ونفهم التكلفة.

1️⃣ الحصول على Claude API Key

خطوات الحصول على API Key

1. ادخل على: console.anthropic.com
2. سجّل دخول أو اعمل حساب جديد
3. اضغط "Get API Keys" من القائمة الجانبية
4. اضغط "Create Key"
5. سمّي المفتاح: PowerAutomate_Key
6. انسخ المفتاح واحفظه في مكان آمن

️ مهم: المفتاح ده زي الباسورد — متشاركهوش مع حد!

2️⃣ Cost Warning — تحذير التكلفة

💰 حساب تكلفة Claude API شهرياً

سعر Claude API:
- Input: $3 لكل مليون توكن
- Output: $15 لكل مليون توكن

مثال: موظف فواتير بيستخدم الـ API يومياً:
- 10 طلبات يومياً × 500 توكن = 5,000 توكن يومياً
- في الشهر (22 يوم) = 110,000 توكن
- التكلفة: ~$0.33 شهرياً (أقل من دولار!)

💡 الخلاصة: التكلفة ضئيلة جداً مقابل التوفير في الوقت
لكن لازم تكون aware بيها

3️⃣ API Rate Limits

⚠️ API Rate Limits

في الحساب المجاني:
- 50 طلب في الدقيقة
- 100,000 توكن في اليوم

في الحساب المدفوع:
- 1000 طلب في الدقيقة
- غير محدود تقريباً

💡 50 طلب/دقيقة = كافي جداً لشغل موظف واحد

Demo: بناء الـ Flow الذكي (جزء 1)

Trigger + HTTP Action — ٥ دقيقة

دلوقتي هنفتح Power Automate ونبني الـ Flow الذكي خطوة بخطوة. هنقسمه لجزئين عشان يكون أوضح.

مثال ٢

Flow: تلخيص شكاوى العملاء وحفظها في Excel

الهدف: إيميل شكوى يوصل → Claude يلخصه ويستخرج رقم الحساب → البيانات تتحفظ في Excel.

الخطوة 1: Trigger

  1. افتح Power Automate → Create → Automated cloud flow
  2. Trigger: When a new email arrives (V3)
  3. اضبط: Folder = Inbox، Subject Filter = شكوى

الخطوة 2: HTTP Action (القلب)

  1. New step → ابحث عن HTTP
  2. Method: POST
  3. URI: https://api.anthropic.com/v1/messages
  4. Headers:
    • x-api-key: [API Key بتاعك]
    • anthropic-version: 2023-06-01
    • content-type: application/json
  5. Body: انسخ الـ JSON ده:
{ "model": "claude-3-5-sonnet-20241022", "max_tokens": 1024, "messages": [ { "role": "user", "content": "استخرج من الإيميل التالي: رقم الحساب، نوع الشكوى، والملخص في جملة واحدة. ارجع النتيجة بصيغة JSON فقط. الإيميل: @{triggerOutputs()?['body/body']}" } ] }
Checkpoint 1: اختبر الـ HTTP Action
  • احفظ الـ Flow
  • ابعت إيميل تجريبي فيه "شكوى"
  • افتح Run history
  • شوف الـ HTTP Action: نجح (200 OK) ولا فشل؟
سر الـ Prompt جوا الـ JSON

لاحظ إني قلت لـ Claude: "ارجع النتيجة بصيغة JSON فقط". ده مهم جداً عشان الـ Parse JSON يقدر يفك شفرة الرد ويحطه في أعمدة Excel.

Demo: Parse JSON + Excel (جزء 2)

تحويل JSON لبيانات + حفظ في Excel — ١٥ دقيقة

دلوقتي Claude رد علينا بـ JSON. لكن الـ JSON ده نص — محتاجين نحوله لبيانات نقدر نستخدمها في Excel. هنا بييجي دور Parse JSON.

شرح Parse JSON

Parse JSON بيحول النص لبيانات

قبل Parse JSON (نص):
{"رقم_الحساب": "12345", "نوع_الشكوى": "عطل في العداد", "الملخص": "العميل يبلغ عن انقطاع المياه"}

بعد Parse JSON (بيانات):
- رقم_الحساب = "12345"
- نوع_الشكوى = "عطل في العداد"
- الملخص = "العميل يبلغ عن انقطاع المياه"

الـ Parse JSON بياخد الـ JSON ويحط كل قيمة في متغير تقدر تستخدمه في Excel.

مثال ٣

إضافة Parse JSON + حفظ في Excel

الخطوة 3: Parse JSON

  1. New step → Parse JSON
  2. Content: اختر Body من HTTP Action
  3. Schema: اضغط "Generate from sample" وحط الرد التجريبي من Claude
  4. الـ Schema هيتولد تلقائياً

الخطوة 4: Add row to Excel

  1. New step → Add a row into a table (Excel Online)
  2. اختار الملف والجدول
  3. في الأعمدة: استخدم Dynamic Content من Parse JSON
    • رقم الحساب: اختر من Parse JSON
    • نوع الشكوى: اختر من Parse JSON
    • الملخص: اختر من Parse JSON
    • التاريخ: utcNow()
    • الحالة: اكتب جديد
✅ النتيجة: Flow ذكي شغال. إيميل يوصل → AI يفهمه → Excel يتحدث ببيانات نظيفة ومنظمة.
نصيحة عملية

لو Parse JSON فشل، غالباً السبب إن Claude رد بـ Text عادي بدل JSON. الحل: عدّل الـ Prompt وقله "ارجع JSON فقط بدون أي نص إضافي".

تمرين عملي: Flow الذكاء

التطبيق الفعلي — ١ دقيقة

دلوقتي دورك. هتبنّي Flow ذكي لسيناريو تاني من شغلك. استخدم نفس البنية وعدّل بس الـ Prompt.

مثال ٤

السيناريو: تحليل تقارير الصيانة

المطلوب: تقرير صيانة يوصل على إيميل → Claude يستخرج: اسم الفني، نوع العطل، الوقت المستغرق → يحفظ في Excel.

التعديل المطلوب في الـ JSON Body:

{ "model": "claude-3-5-sonnet-20241022", "max_tokens": 1024, "messages": [ { "role": "user", "content": "من تقرير الصيانة التالي، استخرج: اسم الفني، نوع العطل، الوقت المستغرق بالدقائق. ارجع النتيجة بصيغة JSON فقط. التقرير: @{triggerOutputs()?['body/body']}" } ] }
✅ النتيجة المتوقعة: Flow بيحلل تقارير الصيانة ويحط البيانات في Excel أوتوماتيك.

تدريب ٢ — عدّل الـ Prompt

١٠ دقايق
  1. افتح الـ Flow اللي بنيته في التدريب السابق
  2. عدّل الـ Prompt جوا الـ JSON Body ليناسب سيناريو شغلك
  3. اختبر الـ Flow ببيانات حقيقية
  4. تأكد إن الـ Parse JSON شغال والبيانات بتدخل Excel صح

قاموس المصطلحات + API Limits

مرجع سريع — ١٥ مصطلح
API
واجهة برمجية
الباب اللي التطبيقات بتتكلم من خلاله مع بعض.
HTTP Action
إجراء HTTP
Action في Power Automate بيبعت Request ويستقبل Response.
JSON
لغة نصية منظمة
اللغة اللي الـ APIs بتتكلم بيها. سهلة القراءة للإنسان والآلة.
Endpoint
نقطة النهاية (URL)
العنوان اللي بنبعتهله الـ Request (مثل: api.anthropic.com).
API Key
مفتاح الدخول
كود سري بيثبت إنك مخول تستخدم الـ API.
Request
الطلب
الرسالة اللي بنبعتهالـ API (فيه Headers و Body).
Response
الرد
الرسالة اللي الـ API بيرجعلهالنا (فيه Status Code و Body).
Status Code
كود الحالة
رقم بيوضح نجاح أو فشل الطلب (200 = نجاح، 400 = خطأ).
Headers
الترويسة
معلومات تقنية بتتبعت مع الـ Request (مثل: API Key).
Body
المحتوى
البيانات الفعلية اللي بنبعتهالـ API (الـ JSON).
Parse JSON
تحليل JSON
Action في Power Automate بيحول نص JSON لبيانات قابلة للاستخدام.
Schema
المخطط
الهيكل اللي بنقوله لـ Parse JSON يتوقعه في الـ Response.
Dynamic Content
المحتوى الديناميكي
بيانات من Actions سابقة متاحة للاستخدام في Actions لاحقة.
Token (API)
وحدة قياس الاستخدام
الـ API بيحسب التكلفة بالتوكنز (كل 4 حروف ≈ توكن واحد).
Rate Limit
حد المعدل
أقصى عدد طلبات تقدر تبعتها في الدقيقة/الساعة.

التدريب الختامي والواجب البيتي

تطبيق شامل — آخر ١٠ دقايق

تدريب ختامي — Flow ذكي من شغلك

١٠ دقايق
  1. فكّر في مهمة في شغلك بتاخد منك وقت في القراءة والتحليل
  2. ارسم الـ Flow: Trigger → HTTP Action (Claude) → Parse JSON → Excel
  3. اكتب الـ Prompt المناسب جوا الـ JSON Body
  4. ابني الـ Flow واختبره ببيانات حقيقية
  5. تأكد إن البيانات بتدخل Excel منظمة ونظيفة

الواجب البيتي — سلّميه في المحاضرة الجاية

  • ابنِ Flow ذكي حقيقي من شغلك (مش التدريب — Flow فعلي)
  • اختبره بـ 3 إيميلات/ملفات مختلفة
  • صور الـ Run history لكل اختبار (نجح/فشل)
  • احسب التكلفة الشهرية المتوقعة للـ API
  • اكتبلي إيه الـ Prompt اللي استخدمته وإيه النتائج
  • جهّز نفسك للمحاضرة الجاية: التحسين والتأمين والمشاريع الختامية

أسئلة المراجعة الذاتية — نظرية

١. ما الفرق بين الأتمتة العادية والأتمتة الذكية؟
٢. ما هو الـ HTTP Action وإيه وظيفته في Power Automate؟
٣. ليه لازم نطلب من Claude يرجع النتيجة بصيغة JSON فقط؟
٤. ما معنى Status Code 401 و 400؟
٥. ما هو الـ Parse JSON وليه مهم بعد الـ HTTP Action؟

أسئلة المراجعة الذاتية — عملية

١. اكتب JSON Body لاستخراج "اسم العميل" و"رقم الهاتف" من إيميل.
٢. عدّل Prompt عشان يصنّف الإيميلات لـ (عاجل، متوسط، عادي).
٣. ابنِ Flow يلخص تقارير الصيانة ويحفظها في Excel.
٤. اصلح Flow فشل بسبب إن Claude رد بـ Text مش JSON.
٥. اختبر Flow بـ API Key غلط واقرأ رسالة الخطأ.
استعداد للمحاضرة الجاية (الختامية)

في المحاضرة السادسة، هنحوّل الـ Flows لنظام احترافي مؤمَّن. هنضيف Conditions، Error Handling، ونبني 3 مشاريع ختامية شغّالة. جهّز الـ Flows اللي بنيتها وكن مستعد!
